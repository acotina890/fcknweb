¿Qué es una página web?

Es un conjunto de etiquetas organizadas en archivos que permiten reflejar la información.

Un sitio web, es un conjunto de archivos donde cada uno está relacionado con otro a través de links (uniones entre páginas). Los archivos tienen que seguir una serie de reglas para que puedan ser visualizados por cualquier navegador existente.

Los archivos HTML son archivos de texto en donde incorpora/n las etiquetas siguiendo un orden propio del lenguaje.

Lenguajes de programación

Los lenguajes de programación se determinan según el nivel de abstracción, la forma de ejecución y el paradigma de programación que posee cada uno.

1) Mediante un programa que va adaptando las instrucciones conforme son encontradas, se lo llama interpretar y a los programas se los conoce como interpretes.
2) Traduciendo este programa al programa equivalente escrito en lenguaje de máquina, se lo llama compilar y al traductor se lo conoce como compilador.

En informática, la programación es un proceso por el cual se escribe -en un lenguaje de programación-, se prueba, depura y se mantiene el código fuente de un programa.
(Los programas son elementos que forman el software, el conjunto de instrucciones que ejecuta el hardware de una computadora para realizar una tarea determinada.)

A medida que pasó el tiempo, las tecnologías fueron desarrollándose y surgieron nuevos problemas para dar solución. Esto dio lugar a desarrollar lenguajes de programación para la web dinámica, que permitieran interactuar con los usuarios y utilizaran sistemas de bases de datos.

HTML
Acrónimo en inglés HyperText Markup Language, en castellano Lenguaje de Marcas Hipertextuales. Es un lenguaje estático para el desarrollo de sitios web, desarrollado por el World Wide Web Consortium (W3C) y los archivos pueden tener las extensiones htm, html. [Ventajas / Desventajas]

Javascript
Es un lenguaje interpretado, no requiere compilación. Fue creado por Brendan Eich, en la empresa Netscape Communications y es utilizado principalmente en páginas web. Es similar a Java, aunque no es un lenguaje orientado a objetos, el mismo no dispone de herencias. [Ventajas / Desvantajas]

PHP
Surgió en 1995, desarrollado por PHP Group y utilizado para la creación de sitios web. Es un acrónimo recursivo que inicialmente se llamó Personal Home Page, y significa "PHP Hypertext Pre-processor".
Es un lenguaje de script interpretado en el lado del servidor utilizado para la generación de páginas web dinámicas, embebidas en páginas HTML y ejecutadas en el servidor. No necesita ser compilado para ejecutarse. Para su funcionamiento necesita tener instalado Apache o IIS con las librerías de PHP. La mayor parte de su sintaxis ha sido tomada de C, Java y Perl con algunas características específicas. Los archivos cuentan con extensión php.
[Ventajas / Desventajas]

ASP
Es una tecnología del lado de servido desarrollado por Microsoft para el desarrollo de sitios web dinámicos. En inglés, Active Server Pages. Este lenguaje fue liberado por Microsoft en 1996. Existen varios lenguajes para crear páginas ASP, el más utilizado VBScript, nativo de Microsoft. Se puede hacer también en Perl y JScript (no Javascript). El código ASP puede ser insertado junto con el HTML. Los archivos cuentan con la extensión asp.
[Ventajas / Desventajas]

ASP.NET
Es un lenguaje comercializado por Microsoft y es el sucesor de ASP. Fue lanzado mediante una estrategia denominada .NET y desarrollado precisamente para resolver las limitantes que brindaba su antecesor. Para desarrollar web sencillas o grandes aplicaciones. Se puede utilizar C#, VB.NET o J#. Los archivos cuentan con la extensión aspx. Para el funcionamiento de las páginas se necesita tener instalado IIS con el framework .Net.
[Ventajas / Desventajas]

JSP
Es un lenguaje para la creación de sitios web dinámicos, acrónimo de Java Server Pages. Orientado a desarrollar páginas web en Java. Es un lenguaje multiplataforma, para ejecutarse del lado del servidor. Fue desarrollado por Sun Microsystems y comparte ventajas similares al ASP.NET. Posee un motor de páginas basado en los servlets de Java. Para su funcionamiento se necesita tener instalado un servidor Tomcat.
[Ventajas / Desventajas]

Python
Creado en el año 1990 por Guido Van Rossum. Es el sucesor del lenguaje ABC. Es comparado habitualmente con Perl y sus usuarios lo consideran como un lenguaje más limpio para programar. Permite la creación de todo tipo de programas incluyendo los sitios web, no necesita ser compilado por lo que es interpretado. Es un lenguaje multiparadigma, lo cual fuerza a que los programadores adopten por un estilo en particular.
[Ventajas / Desventajas]

Ruby
Es un lenguaje interpretado de muy alto nivel y orientado a objetos. Desarrollado en 1993 por el programador japonés Yukihiro Matz Matsumoto. Su sintaxis está inspirada en Python y Perl. Es distribuido bajo licencia de software libre (Opensource). Es un lenguaje dinámico para una programación rápida y sencilla.
[Ventajas / Desventajas]

ColdFusion
Es un entorno de desarrollo dinámico y un servidor Web que permite trabajar con distintos lenguajes como ASP, PHP, JSP, etc. Integra el ColdFusion Markup Language, un lenguaje creado por Macromedia (ahora Adobe) cuyo funcionamiento se basa en etiquetas especiales integradas sobre el código HTML. Es una plataforma que se puede ejecutar de forma concurrente con la mayoría de servidores Web de Windows, Mac, Linux y Solaris.
[Ventajas / Desventajas]

Lenguajes dinámicos vs estáticos

Mientras HTML solo es posible crear sitios web estáticos, mediante los lenguajes web se pueden crear sitios dinámicos, es decir, que pueda tener cambios frecuentes. Para crear una página dinámica, no basta con programar en HTML ya que es muy limitado, por lo que es necesario combinarlo con otros como Perl, PHP, JSP, ASP.NET, JavaScript, Java, etc. La generación del contenido dinámico puede suceder en el servidor o cliente, empleándose lenguajes distintos en cada caso. A la combinación de estos elementos se lo conoce como HTML dinámico (DHTML, Dynamic HTML).

5 claves para un sitio web dinámico

Interactividad
Debe permitir al visitante encontrar su propio camino hasta encontrar la información que está buscando, en lugar de sentarse a recibir la información. Tomando decisiones y realizando, tareas los visitantes se involucran en múltiples niveles creando así una experiencia más robusta.

Sincronización
Debe reunir la información y actividades relevantes de distintas fuentes, ya sea directamente o a través de vínculos, en el momento en el que los necesita.

Flexibilidad
Debe proporcionar al visitante formas de encontrar la información o realizar tareas para que pueda escoger el método que mejor se ajuste a sus necesidades. Por ejemplo, presentarle menús y campos de búsqueda para que navegue por el sitio.

Adaptabilidad
Se ajusta para ofrecer servicio a las necesidades de cada visitante. A veces se hace en el servidor a través de personalización de contenido pero los diseñadores pueden hacer mucho más para acomodar al visitante sin requerir que cargue una nueva página. Por ejemplo, los menús se deslizan dentro y fuera pero pueden ocultarse para maximizar el área de visualización.

Actividad
Utiliza movimientos y sonido para atraer la atención sobre los cambios en pantalla, presentados en forma de animación. Es uno de los principios dinámicos más difíciles de aplicar con eficacia. Una página web estática sirve a cualquier visitante de la misma manera, por el contrario en una dinámica, a pesar que el contenido del servidor es el mismo, en vez de ser solo HTML tiene código dinámico. Se debe instalar un software especial en el servidor para crear una página web dinámica, donde los ficheros se guardan con extensiones especiales para indicarle al servidor que necesitan un procesado adicional antes de enviarlos al cliente (como que se inserten datos desde la bd). Hay muchos lenguajes dinámicos: PHP, Ruby, Python, ASP.NET y ColdFusion, todos estos tienen más o menos las mismas capacidades, como hablar con bds, validar información introducida en formularios, etc.

¿Por qué HTML?

HTML se usa también para la lectura de documentos en internet desde diferentes equipos gracias al protocolo HTTP, que permite acceder de forma remota a documentos almacenados en una dirección específica de la red denominada URL.
La World Wide Web (WWW), es la red mundial formada por todos los documentos (llamados "páginas web") conectados entre sí por hipervínculos.
A menudo las páginas web se organizan alrededor de una principal que actúa como eje central para buscar otras con hipervínculos. Este grupo de páginas unidas y centradas alrededor de una se llama sitio web.
La web es un amplio archivo dinámico compuesto de una gran variedad de sitios y que permite acceso a páginas.

¿Cuál es la función de un maquetador?
El maquetador debe convertir un diseño hecho por los diseñadores, en páginas estáticas XHTML/CSS con las imágenes adaptadas para la web, para que luego sean programadas por los desarrolladores web.
¿Y la función de un desarrollador web?
Podríamos ver el perfil como un contenedor de perfiles, hace de todo: maqueta, programa en cliente y servidor, diseños, etc. Como el mismo nombre induce a entender, es un experto en desarrollo web y debe dominar todas las tecnologías que convergen en una web.

Navegadores en la web
Un navegador es un programa que permite ver la información que contiene una página web, ya se encuentre en un servidor de la WWW o local. Interpreta el código, HTML gralmente., en el que está escrita la página y lo presenta en pantalla permitiendo al usuario interactuar y navegar. La funcionalidad básica de un navegador, es permitir la visualización de documentos de texto, posiblemente recursos multimedia incrustados. El seguimiento de enlaces de una página a otra, se llama navegación donde se origina el nombre navegador.

¿Qué son las etiquetas?

Son elementos dentro del código que funcionan como una especie de marca que le otorga a la sección encerrada una característica específica (como color, fuente, tamaño, tipo de elemento, etc).

ejemplo de etiqueta

La mayoría de las directivas aparecen en parejas y rodean un bloque de texto u otras directivas. Están colocada entre los símbolos menor y mayor. La de inicio de cada par indica al navegador donde comienza la instrucción, y la de final donde termina. Las de cierre siempre incluyen una barra después del símbolo menor.

¿Qué son las etiquetas semánticas?
Son aquellas que dan un significado a las partes del documento. Por decirlo de otra forma, nos indican qué es el contenido que engloban y cuál es su relación con el conjunto de elementos del documento.

La primera línea de código es la declaración de un documento HTML. Indica qué versión se utiliza para que los navegadores puedan aplicar la gramática y sintaxis correspondiente. La versión 5 simplifica mucho esta directiva, las anteriores usaban declaraciones más complicadas.
